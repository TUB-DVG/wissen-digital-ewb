{% translate "Technische Standards" %}

    {% blocktranslate %}Normen und Protokolle spielen eine zentrale Rolle im Kontext der Gebäudeautomation und auch in der Energiewende im Gebäubesektor. Mit Hilfe von Normen sollen Standards und Richtlinien festgelegt werden, die sicherstellen, dass Gebäude entsprechend der neuesten technologischen Entwicklungen und nachhaltigen Prinzipien gestaltet, gebaut und betrieben werden. Diese Normen decken Aspekte wie Wärmedämmung, Energieeffizienz von Heizungs-, Lüftungs- und Klimatechnik sowie den Einsatz erneuerbarer Energiequellen ab. Sie dienen als Leitfaden für Architekt*innen, Bauherr*innnen und Fachleute der Baubranche, um Gebäude zu schaffen, die den höchsten Standards der Energieeffizienz entsprechen und so zur Reduzierung des Energieverbrauchs und der Umweltauswirkungen beitragen.{% endblocktranslate %}

    {% blocktranslate %}Mit Hilfe von moderner Gebäudeautomation und deren intelligenten Steuerung und Überwachung von Gebäudetechnik und -systemen kann der Energieverbrauch optimiert werden. Dazu sind Protokolle notwendig, die spezifische Kommunikations- und Steuerungsstandards definieren, um eine nahtlose Interaktion zwischen verschiedenen Komponenten der Gebäudeautomatisierung zu gewährleisten. Diese Protokolle ermöglichen die Vernetzung von Beleuchtungs-, Heizungs-, Kühlungs-, Lüftungs- und Sicherheitssystemen, um den Energieverbrauch zu regulieren und den Komfort für die Bewohner*innen zu maximieren. Sie sorgen dafür, dass Gebäude effizient arbeiten, indem sie Daten sammeln, analysieren und auf Grundlage von vordefinierten Algorithmen automatisierte Aktionen auslösen. Normen und Protokolle sind somit Schlüsselelemente, die dazu beitragen, die Ziele der Energiewende im Gebäudesektor und der intelligenten Gebäudeautomation erfolgreich umzusetzen.{% endblocktranslate %}

{% translate "Normen" %}

{% blocktranslate %}Auf dieser Seite befinden sich unterschiedliche technische Normen, die im Herbst 2022 erfasst worden sind. Diese sind durch Recherche in Softwarekatalogen, Normendatenbanken und in Forschungsprojekten der Energiewendebauen Projekte erfasst worden. Es handelt sich um deutsche, österreichische, europäische oder internationale Standards und Richtlinien für die Planung, den Bau, und/oder den Betrieb von Lüftungs-, Trinkwasser-, Warmwasser-, Entwässerungs-, Heizungs-, Kühl-, und sonstiger Energiesysteme in Gebäuden. Andere technische Standards wie Schallschutz, Beleuchtung und Gasinstallation sind ebenfalls enthalten. Für jede Norm wurden die folgenden Aspekte erfasst:{% endblocktranslate %}

  • {% blocktranslate %}Name und vollständiger Titel der Norm{% endblocktranslate %}

  • {% translate "Quelle" %} – {% blocktranslate %}woher kann die Norm bezogen werden?{% endblocktranslate %}
  • {% translate "Link" %} – {% blocktranslate %}ein Link, über den die Norm abgerufen (ggf. gekauft) werden kann.{% endblocktranslate %}

{% translate "Protokolle"|capfirst %}

{% blocktranslate %}Auf der Seite sind verschiedenen Protokolle, die im Sommer 2023 erfasst worden sind, ausführlich beschrieben. Diese sind durch Recherche in der entsprechenden Fachliteratur, in Dokumentationen von Hard- und Software und in Unterlagen aus den Forschungsprojekten der Energiewendebauen zusammengetragen worden. Wir haben 12 häufig verwendete und moderne Protokolle ausgewählt und für diese die folgenden Eigenschaften zusammengefasst:{% endblocktranslate %}

  • {% translate "Name des Protokolls" %}
  • {% translate "Übertragungsmethoden" %} – {% blocktranslate %}wie werden Daten zwischen den verschiedenen Komponenten und Geräten innerhalb des Systems übertragen (verkabelt, drahtlos oder verkabelt & drahtlos)?{% endblocktranslate %}
  • {% translate "Unterstützte Übertragungsmedien" %} – {% blocktranslate %}wie werden Signale übertragen?{% endblocktranslate %}
  • {% translate "Zugehörige Standards" %} – {% blocktranslate %}spezifische Standards, denen jedes Protokoll zugeordnet werden kann.{% endblocktranslate %}
  • {% translate "Open-Source-Status" %} – {% blocktranslate %}ist das Protokoll öffentlich und frei verfügbar?{% endblocktranslate %}
  • {% translate "Lizenzgebühr" %} – {% blocktranslate %}ist eine Gebühr zur Abdeckung der Kosten für die Nutzung und/oder die Zertifizierung fällig?{% endblocktranslate %}
  • {% translate "Netzwerktopologie" %} – {% blocktranslate %}was ist die physische und logische Anordnung von Geräten und Komponenten in einem Netzwerk?{% endblocktranslate %}
  • {% translate "Implementierte Sicherheitsmechanismen" %} – {% blocktranslate %}ob und welche Mechanismen sind im Protokoll implementiert, um die Datensicherheit zu gewährleisten?{% endblocktranslate %}
  • {% translate "Bandbreite" %} – {% blocktranslate %}die Kapazität des Kommunikationskanals, Daten zwischen den verschiedenen Komponenten des Systems zu übertragen.{% endblocktranslate %}
  • {% translate "Frequenz" %} – {% blocktranslate %}die Frequenz, die Wellenlänge oder die Trägerfrequenz, die von den drahtlosen Kommunikationsgeräten im System verwendet wird.{% endblocktranslate %}
  • {% translate "Reichweite" %} – {% blocktranslate %}die maximale Entfernung, über die die Kommunikation zwischen den verschiedenen Geräten und Komponenten des Systems aufrechterhalten werden kann, ohne dass die Signalstärke zu schwach wird.{% endblocktranslate %}
  • {% translate "Implementierte OSI-Schichten" %} – {% blocktranslate %}die Anwendung der Schichten des OSI-Referenzmodells für die Kommunikation zwischen den verschiedenen Komponenten des Systems.{% endblocktranslate %}
  • {% translate "Ebenen der Gebäudeautomation" %} – {% blocktranslate %}die hierarchische Struktur eines Gebäudeautomation Systems.{% endblocktranslate %}
  • {% translate "Datenmodell Architektur" %} – {% blocktranslate %}wie werden Informationen oder Attribute zu einem Objekt hinzugefügt?{% endblocktranslate %}
  • {% translate "Discovery" %} – {% blocktranslate %}ob das Protokoll die „Discovery“ Funktion verfügt, um Geräte bei der Datenübertragung automatisch zu identifizieren?{% endblocktranslate %}
  • {% translate "Multi Master" %} – {% blocktranslate %}können mehrere Mastergeräte zeitgleich agieren?{% endblocktranslate %}
  • {% translate "Priorität" %} – {% blocktranslate %}Vorgehen, wie die Änderungen oder Aktualisierungen vorgenommen werden.{% endblocktranslate %}
  • {% translate "Typische Anwendung" %} – {% blocktranslate %}ggf. Beispielhafte Anwendung.{% endblocktranslate %}
